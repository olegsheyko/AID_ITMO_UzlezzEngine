# Отчет по лабораторной работе

## Тема
Интеграция физики и системы ввода с визуальной отладкой в учебный игровой движок.

## Цель работы
В рамках лабораторной работы требовалось расширить существующий проект движка, добавив:
- базовую физическую систему;
- универсальную систему пользовательского ввода;
- события столкновений;
- визуальную отладку коллайдеров;
- управление 3D-камерой;
- демонстрационную сцену для проверки реализованного функционала.

## Выполненные задачи

### 1. Реализованы физические компоненты
В ECS были добавлены новые компоненты:
- `Rigidbody`
- `Collider`
- `Camera`

Компонент `Rigidbody` хранит скорость, ускорение, массу и флаг использования гравитации.  
Компонент `Collider` используется для описания ограничивающего объема объекта. В текущей реализации поддерживается `Box`-коллайдер.  
Компонент `Camera` хранит параметры камеры и рассчитанные матрицы `view/projection`.

### 2. Реализована система физики
Создана `PhysicsSystem`, которая выполняет:
- применение гравитации к динамическим объектам;
- обновление позиции объектов на каждом кадре;
- проверку столкновений `AABB vs AABB`;
- простое разрешение коллизий через раздвижение объектов;
- коррекцию скоростей после столкновения.

Физика встроена в общий цикл обновления сцены и работает через ECS.

### 3. Реализована система ввода
В проект была добавлена архитектура ввода:
- `KeyCode`
- `IInputHandler`
- `GlfwInputHandler`
- `InputManager`

`InputManager` предоставляет единый API для:
- чтения состояния клавиш;
- чтения состояния кнопок мыши;
- получения позиции и дельты мыши;
- привязки игровых действий к клавишам.

Это позволило убрать прямую зависимость игровой логики от вызовов GLFW.

### 4. Добавлена поддержка действий
Ввод организован через action mapping.  
Примеры действий:
- `MoveLeft`
- `MoveRight`
- `MoveForward`
- `MoveBackward`
- `Jump`
- `ToggleDebug`
- `CameraForward`
- `CameraBackward`
- `CameraLeft`
- `CameraRight`
- `CameraUp`
- `CameraDown`

Игровая логика работает не с конкретными клавишами, а с действиями, что упростило расширение системы ввода.

### 5. Добавлены события столкновений
Реализованы:
- `CollisionEvent`
- диспетчер событий
- публикация событий из `PhysicsSystem`

При столкновении сущностей система отправляет событие, а в `GameplayState` ведется логирование столкновений.  
Для удобства отладки в лог выводятся не только ID сущностей, но и их `Tag`.

### 6. Реализована визуальная отладка коллайдеров
Создан `DebugRenderSystem`, который отображает коллайдеры в сцене.

Особенности реализации:
- debug-отрисовка включается и выключается клавишей `F3`;
- для отображения используются реальные AABB, которыми пользуется физическая система;
- debug-отрисовка использует те же матрицы камеры, что и обычный рендер сцены;
- отображение AABB было переделано с wireframe-куба на набор из 12 реальных ребер, что устранило артефакты с диагоналями.

Для устранения расхождений между физикой и debug был вынесен общий helper `CollisionUtils`, который используется и в `PhysicsSystem`, и в `DebugRenderSystem`.

### 7. Реализовано управление 3D-камерой
Были добавлены:
- компонент `Camera`;
- система `CameraSystem`.

`CameraSystem`:
- считывает команды пользователя из `InputManager`;
- перемещает камеру;
- обрабатывает поворот камеры мышью;
- вычисляет и обновляет матрицы `view` и `projection`.

### 8. Реализована демонстрационная сцена
Подготовлена сцена, содержащая:
- землю;
- центральную платформу;
- управляемый куб;
- статические препятствия;
- несколько динамических объектов, падающих под действием гравитации.

Также был добавлен процедурно генерируемый меш `primitive:cube` с корректными UV-координатами, чтобы избежать проблем с отсутствующей внешней моделью куба и некорректным наложением текстур.

## Дополнительная отладка и исправления
В процессе работы были обнаружены и устранены следующие проблемы:

### 1. Debug-коллайдеры не зависели от камеры
Изначально debug AABB рисовались с захардкоженными матрицами, из-за чего они оставались на экране даже при повороте камеры.  
Проблема устранена: теперь debug использует активную игровую камеру.

### 2. Debug-рамки содержали лишние диагонали
Изначально коллайдер рисовался как wireframe-куб из треугольников.  
Это создавало визуальные артефакты.  
Проблема устранена путем отрисовки только 12 ребер AABB.

### 3. Расхождение между мешем и коллайдером
Для box-коллайдеров был добавлен механизм подгонки под реальные bounds меша, если в сцене указан дефолтный коллайдер.

### 4. Невидимые столкновения в сцене
При отладке выяснилось, что объект `Pyramid` имел некорректный коллайдер и пересекался почти со всей сценой.  
Для проверки он был временно исключен из демонстрационной сцены.

### 5. Исправлен переход из меню в игровую сцену
Было обнаружено, что нажатие `Enter` не всегда переводило приложение из `MenuState` в `GameplayState`.  
Проблема была устранена через корректировку обработки ввода в меню и `InputManager`.

## Управление

### Управляемый объект
- `W / A / S / D` или стрелки — движение
- `Space` — прыжок

### Камера
- `I / J / K / L` — перемещение камеры
- `U / O` — подъем и опускание камеры
- правая кнопка мыши + движение мыши — поворот камеры

### Отладка
- `F3` — включение и выключение отображения коллайдеров
- `Enter` — переход из меню в игровую сцену

## Итог
В рамках лабораторной работы в проект была успешно интегрирована базовая физическая подсистема, система ввода, события столкновений, отладочная визуализация коллайдеров и управление камерой.  
Была подготовлена демонстрационная сцена, позволяющая проверить работу гравитации, движения, столкновений и debug-режима.

Полученная реализация представляет собой рабочую учебную основу для дальнейшего развития движка: можно расширять физику, добавлять новые типы коллайдеров, улучшать систему камеры, развивать событийную модель и усложнять игровые сцены.

## Ограничения текущей реализации
- используется только `AABB`, без поворота коллайдера вместе с мешем;
- отсутствуют `OBB`, `Sphere vs Box`, трение и триггеры;
- разрешение столкновений выполнено в упрощенном виде;
- часть коллайдеров сцены по-прежнему требует ручной настройки для полного совпадения с художественной геометрией.
